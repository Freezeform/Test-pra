Introduction

Conducting search engine reconnaissance for information leakage is a task under the Information Gathering stage of the WSTG that involves using search engines to discover sensitive information that may have been inadvertently exposed on the internet.

Test Objectives

  • Identify what sensitive design and configuration information of the application, system, or organization is exposed directly (on the organization's website) or indirectly (via third-party services).
  • Confidential files or documents that have been inadvertently exposed on the internet
  • Sensitive information that has been cached or indexed by search engines
  • Configuration files or scripts that reveal system details or credentials
  • Old or vulnerable versions of web applications that are still accessible online

How to Test

To automate this task, testers can use specialized search engines or automated tools that are designed to scan the internet for specific types of information. Some examples of these tools include:

  • Shodan: A search engine that indexes information about internet-connected devices and systems, including web applications.
  • Google Dorks: Specialized search queries that can be used to identify specific types of information on the internet.
  • TheHarvester: An open-source tool that can be used to gather email addresses, subdomains, and other information from public sources like search engines and social media.
  • Baidu, China's most popular search engine.
  • Bing, a search engine owned and operated by Microsoft, and the second most popular worldwide. Supports advanced search keywords.
  • binsearch.info, a search engine for binary Usenet newsgroups.
  • Common Crawl, "an open repository of web crawl data that can be accessed and analyzed by anyone."
  • DuckDuckGo, a privacy-focused search engine that compiles results from many different sources. Supports search syntax.
  • Internet Archive Wayback Machine, "building a digital library of Internet sites and other cultural artifacts in digital form."

Example for Pentester :

Google Search :

  1. Searching for confidential files: intitle:"index of" "passwords.txt" OR "passwords.doc"
  2. Searching for login pages: intitle:"index of" "passwords.txt" OR "passwords.doc"
  3. Searching for outdated software: intitle:"index of" "phpmyadmin" OR "mysql" OR "wordpress"
  4. Finding email addresses: "email" site:targetwebsite.com
